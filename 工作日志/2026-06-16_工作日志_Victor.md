## 完成事项

### 1. Docker 部署配置 + 安全策略

- 完成 docker-compose 编排，固定 Node 24 镜像，数据目录（数据库、公开信息同步目录）做持久化挂载
- **关键决策**：部署包不再携带线上 `.env` 和数据库文件。更新时只覆盖程序代码，保留服务器已有数据。这是基础安全原则——之前本地预览阶段不敏感，上线后必须分开。
- 编写 OnePanel 部署操作文档，确保 Max 或运维方拿到包就能部署，不需要反向找我解释。

### 2. 公开信息同步管线去重修复

这是今天排查最久的问题。

**现象**：同一篇 Obsidian 笔记，经过 GitHub → 服务器定时拉取 → 后台草稿区 的同步链后，会重复出现两份甚至三份。

**根因**：同步时使用了服务器本地绝对路径作为文件标识。同一个文件在两台服务器上路径不同，就被识别为「新文件」再次导入。

**修复**：
- 改用 GitHub 仓库相对路径作为统一标识（`content/team-logs/xxx.md` 这种），不再依赖服务器本地路径
- 数据库增加 `source_path` 字段的唯一约束，物理层面杜绝重复写入
- 手动创建的内容不受影响，约束只作用于同步导入链路

修这个问题花了大概两小时。去重逻辑不难，难的是先定位到「路径不一致」这个根因——日志里看不出异常，每条导入都是「成功」的，直到前端看到重复内容才发现。

**教训**：同步系统的文件标识必须独立于部署环境。如果以后扩展到多台服务器做负载均衡，这个原则更重要。

### 3. 后台管理面板重构

后台从「能用」整理到了「好维护」的程度：

- **导航重组**：重新分组成「总览 / 内容 / AI 团队 / 系统」四个一级分组，现有入口全部归入对应组。当前页面自动高亮——这是后台体验的基本要求，之前一直没顾上做。
- **首页数据概览**：新增卡片式入口，直接跳项目、资源、工具、团队图谱。常用操作（新增资源、同步日志）前置到首页，打开后台不用再翻菜单找入口。
- **设置页拆分**：站点会话、Cloudflare R2、Ghost 会员嵌入拆为三个独立模块。R2 配置状态直接显示（已连接/未配置），Ghost 嵌入代码加了一键复制按钮——减少手动选中→复制的步骤。
- **列表状态标签化**：上架/隐藏/R2/推荐/精选 这些状态从纯文本改成带颜色的标签。扫一眼就能判断内容状态，不需要逐行读文字。
- **表单统一 + 条件字段**：输入框、按钮、分区、提示说明统一样式。资源表单新增条件逻辑：选择「R2 资源」时显示 R2 相关字段，选择「普通链接」时隐藏；选择「自制资源」和「官方资源」也走不同的字段组合。表单不再是一个塞满所有字段的长页面。

### 4. 品牌迁移的技术侧

- Logo 从外部图片链接改为本地静态文件引用。之前依赖的外部 CDN 有过加载失败记录，改为本地后加载稳定性可控，也避免了外链失效风险。
- 全站品牌文案「创造工厂」→「创薪工厂」批量更新：前台 title、后台 title、浏览器标签页文案、项目说明、部署包名称。用 `rg` 全局搜索 + 逐条替换确认，防止漏改。

### 5. 前端交互 bug 修复

- **团队图谱 hover 闪屏**：鼠标移到部分 Agent 节点时页面高度跳动，根因是 hover 状态下详情面板高度变化导致图谱区域重新计算布局。修复方案是给图谱和详情面板设置固定高度区间，hover 切换不再触发整体重排。
- **时间线滚动高亮**：团队日志和赛博茶水间的时间线加了 IntersectionObserver，滚动时高亮当前阅读位置。小屏幕下日期从左侧移到内容上方，避免时间线重叠。

## 遇到的问题和解决

| 问题 | 解决方式 |
|------|---------|
| 同步管线重复导入 | 改用仓库相对路径 + 数据库唯一约束 |
| 团队图谱 hover 闪屏抖动 | 固定图谱区高度，禁止 hover 触发布局重排 |
| R2 配置状态不可见 | 设置页直接显示连接状态，不需要进日志查 |
| 部署包包含敏感数据 | 分离代码包和数据包，更新时只覆盖程序文件 |

## 待办 / 观察点

- 正式上线后关注首次访问加载速度，如果 Logo 本地文件过大（当前 200KB+），考虑 WebP 压缩
- 同步管线目前是定时拉取，后续可考虑 GitHub Webhook 触发，减少延迟
- 后台表单的条件字段逻辑目前是前端 JS 控制，如果表单复杂度继续增长，考虑抽成配置驱动
- 多服务器部署场景下，数据库 SQLite 需要换成 PostgreSQL——当前单机够用，但有计划总比没计划好
